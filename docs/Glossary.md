# Glossary of Terms

- **CP-SAT Solver**: The mathematical optimization engine from Google OR-Tools used to solve the team assignment problem.
- **Docling**: The document processing library used to extract high-fidelity text from PDFs and Docx files.
- **Fuzzy Skill Matching**: A bidirectional substring comparison logic used in the frontend to bridge naming differences in technical terms.
- **HNSW Index**: Hierarchical Navigable Small World index used in PostgreSQL (via pgvector) for fast and memory-efficient semantic vector searches.
- **LangGraph**: The framework used to orchestrate the multi-node AI extraction agent and manage document processing states.
- **Multimodal Extraction**: An AI process that analyzes both text and visual components (like slide images) of a document simultaneously.
- **pgvector**: The PostgreSQL extension that adds native support for vector similarity search.
- **Semantic Similarity**: A measure of technical alignment calculated using 1024-dimensional embeddings and cosine similarity.
- **StandardSkill**: A canonical technical term used for organizational skill normalization and taxonomy management.
- **Truth-Anchored Extraction**: A strict AI protocol that prevents hallucinations by requiring every extracted fact to have verbatim evidence in the source document.
- **UniversalTechnicalProfile**: The standardized Pydantic data model used to store extracted member and project data.
- **watsonx.ai**: The IBM Cloud platform providing foundation models for text reasoning (Llama 3.2) and embedding generation.
